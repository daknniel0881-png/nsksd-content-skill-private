# 日生研NSKSD纳豆激酶 · 自媒体内容工厂

> 品牌：日生研生命科学（浙江）有限公司
> 产品：NSKSD纳豆激酶（日本生物科学研究所生产，日生研为中国总代理）
> 目标：通过自媒体内容吸引美容院老板、养生馆老板、社区门店老板成为分销商
> 内容策略：不直接卖货，通过行业洞察、科学背书、赚钱逻辑建立信任

## 使用方式

本Skill支持5个子命令，对应内容生产的五个阶段。用户可以按顺序走完整流程，也可以单独调用任一阶段。

| 命令 | 阶段 | 说明 |
|------|------|------|
| `/nsksd 选题` | 情报收集+选题生成 | 生成一批选题并按S/A/B分级 |
| `/nsksd 标题 <选题>` | 标题优化 | 为指定选题生成5个标题变体并打分 |
| `/nsksd 大纲 <选题>` | 内容骨架 | 生成完整的文章大纲和写作指引 |
| `/nsksd 文案 <选题>` | 全文撰写 | 基于知识库素材生成完整文案 |
| `/nsksd 复审 <文案>` | 合规复审 | 检查合规红线、通俗度、禁用词、科学准确性 |

无子命令时默认执行"选题"阶段。

## 阶段一：选题生成（`/nsksd 选题`）

### 输入
用户可提供：
- 特定方向（如"围绕FFC大会""针对美容院老板"）
- 热点关键词（如"血管健康""三高"）
- 不提供任何输入也可以，Skill会自动从知识库和热点中生成

### 执行流程

**第一步：素材扫描**
1. 读取 `references/knowledge-base.md` 获取知识库核心素材
2. 读取 `references/topic-library.md` 获取已有选题库
3. 如果用户提供了热点方向，结合热点生成；否则从知识库素材中碰撞生成

**第二步：选题生成**
围绕四条内容线生成选题，配比为：
- **科学信任线（40%）**：临床数据、专家背书、研究成果 → 建立信任
- **健康科普线（30%）**：血管健康、心脑血管、认知障碍 → 吸引泛健康人群
- **品牌故事线（15%）**：日生研历程、行业净化者、差异化 → 打品牌厚度
- **招商转化线（15%）**：门店经营、选品方法论、真实账本 → 精准招商

每个选题必须包含：
```
选题名称：
内容线：科学信任 / 健康科普 / 品牌故事 / 招商转化
核心角度：（一句话说清楚这篇文章要讲什么）
目标人群：美容院老板 / 养生馆老板 / 社区门店老板 / 泛健康人群
目标情绪：XX → XX（如：怀疑→相信）
知识库素材：（列出可用的具体素材）
```

**第三步：选题评审打分**
对每个选题按四个维度打分（每项1-10分）：

| 维度 | 评分标准 |
|------|----------|
| 点击欲望 | 看到这个选题/标题，目标人群想不想点进来？ |
| 接地气程度 | 村口大爷能不能看懂？有没有专业术语障碍？ |
| 招商转化潜力 | 看完能不能引导到分销合作？ |
| 科学可信度 | 有没有临床数据/专家/案例支撑？不能空口说 |

分级标准：
- **S级**（总分≥35）：立刻可以写，预期效果好
- **A级**（28-34分）：值得写，可能需要微调角度
- **B级**（20-27分）：备选，有潜力但需要优化
- **淘汰**（<20分）：不输出

### 输出格式
输出10-15个选题，按S→A→B排列，每个选题附评分和分级。

## 阶段二：标题生成（`/nsksd 标题 <选题>`）

### 输入
一个确定的选题（来自阶段一，或用户自定义）。

### 执行流程

为该选题生成5个标题变体，覆盖不同风格：

**标题公式库**（每次至少使用3种不同公式）：

1. **数字冲击型**：「X人临床数据证明：纳豆激酶不是玄学」
2. **反常识型**：「为什么日本89.4%的人吃纳豆，中国人却在交智商税？」
3. **恐惧驱动型**：「你的客户正在被直播间抢走」
4. **利益承诺型**：「社区门店如何靠一款产品锁定500个家庭？」
5. **提问引导型**：「养生馆老板：你还在靠手艺赚钱吗？」
6. **悬念型**：「4月24日杭州，纳豆激酶行业可能要变天了」
7. **对比型**：「卖纳豆激酶的公司千千万，为什么这家要自己花钱做临床？」

### 标题三原则

1. **村口大爷原则**：不用专业术语，说人话。"颈动脉内膜中层厚度"要翻译成"血管壁变厚"
2. **点击优先原则**：标题决定80%的打开率，宁可花1小时改标题也不要花1小时改内文
3. **人工复核原则**：AI出5个标题，人来挑选和修改。**标题不能100%交给AI**

### 标题禁用清单
- 禁止夸大疗效（"治好""根治""特效""神药"）
- 禁止使用"放大招""炸裂""颠覆""赋能""闭环"等AI味/营销号词汇
- 禁止假冒专家口吻（"医生建议""专家警告"——除非真的是该专家说的）
- 禁止虚假数据
- 禁止使用破折号（——），改用逗号或拆两句

### 输出格式
```
标题1：「xxx」（公式：数字冲击型）
标题2：「xxx」（公式：反常识型）
标题3：「xxx」（公式：提问引导型）
标题4：「xxx」（公式：利益承诺型）
标题5：「xxx」（公式：悬念型）

推荐：标题X（推荐理由）
```

## 阶段三：大纲生成（`/nsksd 大纲 <选题>`）

### 输入
一个确定的选题 + 可选的已选标题。

### 执行流程

1. 读取 `references/knowledge-base.md` 匹配相关素材
2. 按以下万能结构生成大纲：

```
钩子开头（100字以内，制造共鸣或冲突）
    ↓
行业洞察/痛点分析（建立"你懂他"的感觉）
    ↓
解决方案/趋势机会（给出方向）
    ↓
科学背书/数据支撑（建立信任）
    ↓
行动路径/案例验证（降低决策门槛）
    ↓
软性CTA（"了解合作"而非"立即购买"）
```

3. 大纲中每个部分标注：
   - 预计字数
   - 可用的知识库素材（具体到哪份文档的哪个数据点）
   - 写作注意事项

### 输出格式
完整的分段大纲，每段附素材来源和字数建议。总字数建议1500-2500字。

## 阶段四：全文撰写（`/nsksd 文案 <选题>`）

### 输入
一个选题 + 可选的标题 + 可选的大纲。如果没有提供标题和大纲，Skill会先自动执行阶段二和阶段三。

### 执行流程

1. 读取 `references/knowledge-base.md` 获取全部可用素材
2. 读取 `references/compliance.md` 了解合规红线
3. 按大纲结构撰写全文

### 写作风格要求

**核心原则：村口大爷能看懂**

- 口语化为王，像朋友聊天，不像AI写报告
- 永远具体：不说"很多客户"，说"80%的客户"
- 结论先行，再展开论据
- 短中长句交替（23%短句 / 50%中句 / 26%长句）
- 问号远多于感叹号（用追问代替感叹，用留白代替感慨）
- 适度暴露不确定："大概率""说不准""也许吧"

**签名句式**（自然使用，不要每篇都用全）：
- "你会发现" — 引导读者自己得出结论
- "为什么呢？" — 独立成段制造节奏
- "说白了" / "本质上" — 把复杂概念翻译成人话
- "大概率" — 概率性判断，不说绝对话

**禁用词**（写完必查）：
赋能、链路、飞轮、认知升级、颗粒度、抓手、闭环、触达、矩阵、范式、痛点、底层逻辑（单独使用）、护城河（泛化使用）、降维打击、生态位、心智模型

**禁用句式**：
「不是…而是…」「不是…是…」「并非…而是…」等对立否定句式——AI味极浓，改用直接陈述或用"说白了""本质上"引出判断。

**段落节奏**：
- 长文用 ◆ 或 ---- 切段，3-6个独立段落
- 编号列表（1️⃣2️⃣3️⃣）结构化复杂内容
- "……" 留白，话题切换或意犹未尽

**CTA规范**：
- 所有行动引导放在文末，不要在文章中间推销
- 用"了解合作"而非"立即购买"
- 用"想聊聊"而非"立即扫码"

### 数据引用规范

引用知识库数据时必须准确，常用数据点：

| 数据 | 正确表述 | 来源 |
|------|----------|------|
| 1062人临床 | "迄今纳豆激酶领域最大规模临床试验，1062人参与" | 科学循证文献合辑 |
| 斑块改善率 | "颈动脉斑块改善率66.5%-95.4%" | 临床册子 |
| 认知衰退风险 | "视觉空间功能衰退风险降低约65%" | 浙大RCT研究 |
| 专家建议 | "北京神经内科学会等3家学术机构、30余位专家" | 专家建议原文 |
| FU用量 | "10800FU/天（1062人试验）或8000FU/天（浙大RCT）" | 对应研究论文 |
| EFSA安全性 | "欧洲食品安全局2017年认证安全" | 科学循证文献 |

### 输出格式
完整文案（1500-2500字），包含标题、正文、CTA。文末附一段"素材来源清单"供团队核实。

## 阶段五：合规复审（`/nsksd 复审`）

### 输入
一篇已写好的文案（来自阶段四或用户自己写的）。

### 执行流程

读取 `references/compliance.md`，按以下5个维度逐项检查：

**检查项1：合规红线**
- ❌ 是否存在疗效承诺？（"治好""根治""治愈""特效"）
- ❌ 是否暗示可替代药物？（必须明确"不能替代正规治疗"）
- ❌ 是否使用绝对化用语？（"100%有效""包治百病"）
- ❌ 是否虚假宣传？（数据是否有出处？案例是否编造？）
- ❌ 是否违反广告法相关规定？

**检查项2：村口大爷原则**
- 通读一遍，有没有普通人看不懂的专业术语？
- 有没有需要翻译成大白话的段落？
- 开头100字能不能吸引人继续读？

**检查项3：禁用词扫描**
- 扫描禁用词清单（赋能、链路、飞轮、认知升级、颗粒度、抓手、闭环、触达、矩阵、范式、底层逻辑、降维打击、生态位、心智模型、放大招、炸裂、颠覆）
- 扫描禁用句式（「不是…而是…」「并非…而是…」）
- 感叹号数量检查（感叹号应远少于问号）

**检查项4：科学准确性**
- 引用的数据是否与知识库原文一致？
- 有没有夸大研究结论？（例如把"改善风险"说成"治好了"）
- 专家头衔是否准确？

**检查项5：招商自然度**
- CTA是否放在文末？（不能在文章中间就开始推销）
- 招商引导是否自然？（"了解合作"而非"立即购买"）
- 全文读起来像不像广告？（如果像，需要调整）

### 输出格式
```
## 复审报告

### 合规红线：✅ 通过 / ⚠️ 有风险 / ❌ 不通过
（具体问题列表）

### 通俗度：✅ 通过 / ⚠️ 需优化
（需要翻译的术语列表+修改建议）

### 禁用词：✅ 通过 / ⚠️ 发现X个
（发现的禁用词+替换建议）

### 科学准确性：✅ 通过 / ⚠️ 有偏差
（具体偏差和修正建议）

### 招商自然度：✅ 自然 / ⚠️ 偏硬
（修改建议）

### 综合评分：XX/100
### 修订版文案：
（如果有需要修改的地方，直接输出修改后的完整文案）
```

## 关键提醒

1. **纳豆激酶是食品，不是药品**。任何内容都不能暗示治疗效果，必须使用"辅助""养护""改善风险"等表述
2. **NSKSD是具体品牌**，不等于"纳豆激酶"品类。知识库中的临床数据仅针对NSKSD产品，不能推广到所有纳豆激酶
3. **目标读者是门店老板**，不是消费者。内容要讲"赚钱逻辑"，不是讲"产品功效"
4. **村口大爷原则**是最高优先级的写作标准。任何时候在"专业"和"通俗"之间做选择，选通俗
5. **标题永远是第一优先级**。内容素材不缺，关键是"看到标题愿意点进来"
