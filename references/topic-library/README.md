**品牌**：日生研纳豆激酶（**严格用"日生研"三字**，绝不"日升盐""日生盐"）
**版本**：V9.1（2026-04-21）
**目的**：在 S/A/B 级分层之外，**按"内容模块"分块 + 每日搜索整理 + 月度归档**，让选题库越用越厚。

## 一、6 大内容模块（M1–M6）

| 模块 | 目录 | 定位 | 单条产出要求 |

| **M1 · 行业热点** | `M1-industry-news/` | 大健康/医疗/慢病/保健食品的政策与媒体动态 | 每周 ≥ 5 条 |
| **M2 · 纳豆激酶研究** | `M2-nattokinase-research/` | 最新论文/临床试验/专家指南/NSKSD 动态 | 每周 ≥ 5 条 |

| **M6 · 招商场景** | `M6-partner-channels/` | 养生馆/美容院/健康门店/社群的多元招商角度 | 每周 ≥ 3 条 |

> **M1–M6 与"六维坐标系"（`topic-selection-rules.md`）对齐**：M1–M6 是**内容模块**（告诉你"选题从哪个库里拿"），六维坐标系是**维度**（告诉你"这条选题在热度/深度/情绪/人群/形式/痛点上站位"）。两者互补，不是替代。

## 二、M6 招商场景 · 多元角度清单（硬约束）

**曲率铁律**：招商内容 ≠ 单纯"卖给店主、讲利润增长"。M6 的选题必须覆盖以下 **8 个角度**，不得单一：

1. **店主专业形象升级**——让养生馆/美容院老板从"推销者"变"健康顾问"
2. **客户健康咨询能力建设**——教店员如何用科学语言解释纳豆激酶（不违规）
3. **会员裂变与粘性**——血压监测打卡、季度复测、健康档案
4. **产品组合设计**——纳豆激酶配什么（辅酶 Q10、膳食纤维、中医调理），不只推单品
5. **合规风险规避**——养生馆能说什么、不能说什么（广告法/保健食品禁忌语）
6. **跨品类联动**——美容 × 血管健康、中医 × 现代循证
7. **社群运营**——私域打卡、直播科普、医生连线问答
8. **门店动线与话术设计**——免费体测切入、分层转化、复购触发

> 禁止把 M6 压缩成"给养生馆老板的利润增长攻略"这种单一角度。

## 三、搜索整理机制

### 触发方式
```bash
# 搜指定模块
bun scripts/topic-crawler.ts --module M2

# 批量搜所有模块
bun scripts/topic-crawler.ts --all
```

### 关键词映射（见 `scripts/topic-crawler.ts`）
| 模块 | 关键词池 |
|------|---------|
| M1 | 健康行业热点、慢病管理、保健食品、医疗政策、大健康产业 |
| M2 | 纳豆激酶 临床试验、NSKSD、纳豆激酶 心脑血管、专家指南、溶栓 酶 |
| M3 | 高血压管理、动脉粥样硬化、血脂管理、中风预防、健康管理服务 |
| M4 | 纳豆激酶 案例、心脑血管康复故事、医生推荐、健康改善 |
| M5 | 保健食品监管、广告法、执业药师 整治、职业打假 |
| M6 | 养生馆 健康咨询、美容院 健康管理、健康门店 运营、健康社群 |

### 归档规则
- 路径：`{module}/{YYYY-MM}/{YYYY-MM-DD}-{slug}.md`
- **每月 1 号自动新建 `{YYYY-MM}/` 子目录**（脚本负责）
- 旧月份只读归档，不删除
- 单文件模板见本文附录 A

## 四、和现有 S/A/B 分级的关系

分块（M1-M6）= **内容来源**
分级（S/A/B）= **选题价值评分**
维度（六维坐标）= **当日必须覆盖的角度分布**

三者**全部叠加**到 `logs/topic-history.jsonl` 的单条记录上：
```json

```
