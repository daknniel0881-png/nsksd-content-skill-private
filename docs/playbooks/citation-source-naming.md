# Playbook：引用来源命名规范（V10.3）

> **坑位 & 真相**：客户反馈模型在公众号文章里每段后面写"（来源：同上）"，或者用"袁总/老袁"代替人的原名。这两种都违规。V10.3 增设 `citation_check.py` 硬门控拦截。

---

## 核心三条规则

### 规则一：来源用**人的原名**

引用某人的话/数据，必须写全名（姓 + 至少 2 字名），可以带完整头衔。

| ❌ | ✅ |
|---|---|
| 袁总 | 袁岳 |
| 老袁 | 袁岳 |
| 张老师 | 张伟东博士 |
| —— 陈董 | —— 陈文明 |

**合法模板**：
- `姓 + 名`（袁岳、陈文明）
- `姓 + 名 + 头衔`（张文宏教授、王陇德院士、钟南山院士）
- `机构全称`（中华医学会、国家卫健委、日生研研发部）

### 规则二：每次引用都**写清具体来源**

禁止「同上/同前/见上/如上/同 1」。哪怕一篇文章同一个数据出现 3 次，也要每次写全。

| ❌ | ✅ |
|---|---|
| （来源：同上） | （来源：《中国心血管病报告 2024》P142） |
| （来源：同前） | （来源：浙江大学 1062 人 RCT，DOI: 10.xxxx） |
| （来源：见上） | （来源：NSKSD 官网临床研究页） |

**降级写法**（实在找不到原始出处）：
- `（来源：日生研官方资料）` — 仅限品牌自述数据
- `（引自：《中国健康管理与健康产业发展报告（No.6）》专家指引）` — 蓝皮书白名单

### 规则三：用**全角括号** `（来源：XXX）`

半角 `(来源: XXX)` 会被 citation_check 拦下。全角是 `format.py` 的"自动灰化"钩子。

---

## format.py 自动灰化

V10.3 起，`scripts/format/format.py::gray_out_source_citations` 会把所有 `（来源：XXX）`/`（出处：XXX）`/`（引自：XXX）`/`（参考：XXX）` 自动包成：

```html
<span style="color:#9aa0a6;font-size:0.92em">（来源：XXX）</span>
```

渲染后来源字比正文浅、略小（0.92em），和正文拉开视觉层级。

**客户看到的效果**：

```
临床数据显示，颈动脉内膜中层厚度平均下降 0.22mm（来源：Journal
of Cardiovascular Risk, 2023, DOI:10.xxxx）。对三高人群……
                              ↑ 这一段灰色、稍小
```

## citation_check.py 扫描项

| 扫描项 | 违规示例 | 处理 |
|-------|---------|------|
| `honorifics` | `（来源：袁总采访）`/`—— 老袁` | 退回重写 |
| `vague_backref` | `（来源：同上）`/`（来源：同前）` | 退回重写 |
| `half_width_source` | `(来源: xxx)` 半角括号 | 退回重写 |

手动测试：
```bash
python3 scripts/citation_check.py artifacts/<SID>/step3-article.md
# 退码 0=通过 / 1=违规 / 2=文件错
```

触发点：`trigger_watcher.sh` Step 4.5 第三道门控（layout → data_audit → **citation_check** → image_size_check）。

## 常见误伤与 Fix

| 场景 | 是否误伤 | 说明 |
|------|---------|------|
| 正文写"张伟东老师讲过" | 否 | 正文只拦"括号来源内 + 采访者行"里的亲昵称谓。正文文学化表达不拦 |
| "张文宏教授" | 否 | 完整姓 + 名 + 头衔，合法 |
| "老中医" / "老行家" | 否 | "老"非单姓前缀，不误伤 |
| "小王师傅" | 否 | "师"已从词库移除（避免误伤"老师/师傅/工程师"） |

如遇到误伤，改回正规原名即可；如是正则漏拦，补到 `HONORIFIC_PATTERNS`。

## 红线

- ❌ 段落后每次都写「（来源：同上）」→ 100% 被拦
- ❌ 公开内容里写「袁总 / 老袁」→ 100% 被拦
- ❌ 半角 `(来源: xxx)` → 100% 被拦（破坏灰化样式）
- ✅ 全角 `（来源：人原名 + 具体出处）` → 通过 + 自动灰化
