# Playbook：数据与事实验证

> 场景：文章中引用的数据、研究结论、机构名称，在写入文章前必须验证。

---

## 核心原则

**宁可不写，不可捏造。**

每条数据必须过三问：
1. 这个数据有来源 URL 吗？
2. URL 还活着吗（WebFetch 验证）？
3. 有第二个来源交叉印证吗？

---

## 步骤

### 1. 数据初步判断

拿到一条数据点后，先做分类：

| 类型 | 示例 | 处理 |
|------|------|------|
| 已知公开研究 | "浙大 1062 人 RCT" | 找原文 DOI 或 PubMed 链接 |
| 行业统计数据 | "三高患者超 3 亿" | 找卫健委或国家统计局原始报告 |
| 产品数据 | "连续服用 26 周" | 找临床研究论文 |
| 口耳相传 | "据说效果很好" | 直接删除，不入文章 |

### 2. 查找来源

**优先顺序：**
1. PubMed（`https://pubmed.ncbi.nlm.nih.gov/?term=`）
2. 中国知网（`https://www.cnki.net`）
3. 官方机构网站（卫健委、疾控中心、学会官网）
4. 期刊官网（直接搜 DOI）
5. 新闻报道引用的原始研究

**搜索技巧：**
```
# PubMed 搜索纳豆激酶
nattokinase cardiovascular clinical trial

# Google Scholar 搜索浙大研究
浙江大学 纳豆激酶 随机对照试验 2023

# DOI 直接访问
https://doi.org/10.xxxx/xxxxx
```

### 3. WebFetch 验证 URL

```python
# 验证 URL 是否有效（在 Claude Code 里用 WebFetch 工具）
# 检查返回的页面是否包含预期内容

# 人工验证步骤：
# 1. 打开 URL
# 2. 确认页面标题与数据来源匹配
# 3. 在页面内搜索关键数据（Ctrl+F）
# 4. 确认数字精确匹配（不是约等于）
```

### 4. 找不到来源怎么办

**降级写法（按顺序选择）：**

| 情况 | 降级写法 |
|------|----------|
| 有具体研究但找不到原文 | "一项发表在 XXX 期刊的研究发现……（尚缺公开链接）" |
| 有行业通识但无具体来源 | "行业内普遍认为……" |
| 数据来自内部资料 | "根据内部追踪数据，样本量 XX 人" |
| 完全无法核实 | 直接删除该数据点 |

**绝对不做：**
- 不凭记忆写数字（"大约"也不行，有就写精确数，没有就不写）
- 不编造 DOI 或 URL
- 不写"某研究发现"（要写出研究机构或发表期刊）
- 不改数字（1062 人不能写成"约一千人"）

## 数据标注格式

在文章里标注来源：

```markdown
浙江大学连续追踪 1062 人两年，结果发现认知功能衰退风险降低了 65%
（来源：XXX 期刊，2023 年，DOI 链接附 references/data-verification.md）。
```

在 `references/data-verification.md` 维护详细来源表，文章里只写简短标注。

---

## 验证方法

写完文章后运行检查：

```bash
# 检查是否有模糊数字表述
grep -n "约.*人\|大概.*%\|据说\|有研究\|某机构" 文章.md

# 检查是否有未标注来源的数字
grep -n "[0-9]\+%" 文章.md
# 人工逐条确认每个百分比数字是否有来源标注
```

---

## 回查兜底

如果发布后发现数据有误：
1. 立即在公众号后台修改（公众号支持发布后修改正文）
2. 在飞书群发通知，说明修正内容
3. 更新 `references/data-verification.md` 中对应记录，标注"已修正"
4. 如果错误影响较大（涉及医疗建议），[待曲率确认] 是否需要发勘误推文
