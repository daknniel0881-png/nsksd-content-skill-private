# 日生研NSKSD纳豆激酶 · 自媒体内容工厂

> 品牌：日生研生命科学（浙江）有限公司
> 产品：NSKSD纳豆激酶（日本生物科学研究所生产，日生研为中国总代理）
> 目标：通过自媒体内容吸引美容院老板、养生馆老板、社区门店老板成为分销商
> 内容策略：不直接卖货，通过行业洞察、科学背书、赚钱逻辑建立信任

## 使用方式

本Skill支持5个子命令，对应内容生产的五个阶段。用户可以按顺序走完整流程，也可以单独调用任一阶段。

| 命令 | 阶段 | 说明 |
|------|------|------|
| `/nsksd 选题` | 情报收集+选题生成 | 生成一批选题并按S/A/B分级，**含选题合规预检** |
| `/nsksd 标题 <选题>` | 标题优化 | 为指定选题生成5个标题变体并打分，**含标题合规检查** |
| `/nsksd 大纲 <选题>` | 内容骨架 | 生成完整的文章大纲和写作指引 |
| `/nsksd 文案 <选题>` | 全文撰写 | 基于知识库素材生成完整文案，**写作前加载合规清单，写作中实时遵守** |
| `/nsksd 复审 <文案>` | 合规复审 | **20项完整合规检查清单**，逐项审查并出具报告 |
| `/nsksd 合规检查 <文案>` | 独立合规审查 | 对任意文案执行完整合规检查，可独立于其他阶段使用 |

无子命令时默认执行"选题"阶段。

> ⚠️ **合规贯穿全流程**：合规不是"写完再查"，而是"选题就开始考虑"。本Skill在阶段一（选题）、阶段二（标题）、阶段四（撰写）、阶段五（复审）四个阶段都内置了合规机制，确保内容从源头就不会踩线。

## 阶段一：选题生成（`/nsksd 选题`）

### 输入
用户可提供：
- 特定方向（如"围绕FFC大会""针对美容院老板"）
- 热点关键词（如"血管健康""三高"）
- 不提供任何输入也可以，Skill会自动从知识库和热点中生成

### 执行流程

**第一步：素材扫描**
1. 读取 `references/knowledge-base.md` 获取知识库核心素材
2. 读取 `references/topic-library.md` 获取已有选题库
3. 读取 `references/compliance.md` 加载合规红线（**选题阶段就必须加载**）
4. 读取 `references/compliance-checklist.md` 加载选题合规预检清单
5. 如果用户提供了热点方向，结合热点生成；否则从知识库素材中碰撞生成

**第二步：选题合规预检（前置过滤）**

在生成选题时，每个选题必须先过合规预检5项，不通过的选题直接淘汰，不进入评审环节：

| # | 预检项 | 淘汰标准 |
|---|--------|----------|
| 1 | 选题是否暗示纳豆激酶可治疗/治愈疾病？ | 出现"治疗""治愈""根治"等医疗承诺→淘汰 |
| 2 | 选题是否涉及具体疾病诊断？ | 替代医生给出诊断建议→淘汰 |
| 3 | 选题是否与竞品做疗效对比？ | 贬低同行或做功效对比→淘汰 |
| 4 | 选题角度是否可追溯到知识库素材？ | 核心观点无知识库支撑→淘汰 |
| 5 | 选题是否符合"食品"定位？ | 全文语境超出食品范畴→淘汰 |

**选题安全分级**（附在每个选题后面）：
- 🟢 **安全**：品牌故事/行业科普/大会报道/用户教育 → 直接写作
- 🟡 **谨慎**：涉及临床数据/健康功效/专家观点 → 写作时逐句合规审查
- 🔴 **高危**：涉及疾病名称/治疗效果/医学建议 → 必须专人审核，建议法务确认

**第三步：选题生成**
围绕四条内容线生成选题，配比为：
- **科学信任线（40%）**：临床数据、专家背书、研究成果 → 建立信任
- **健康科普线（30%）**：血管健康、心脑血管、认知障碍 → 吸引泛健康人群
- **品牌故事线（15%）**：日生研历程、行业净化者、差异化 → 打品牌厚度
- **招商转化线（15%）**：门店经营、选品方法论、真实账本 → 精准招商

每个选题必须包含：
```
选题名称：
内容线：科学信任 / 健康科普 / 品牌故事 / 招商转化
核心角度：（一句话说清楚这篇文章要讲什么）
目标人群：美容院老板 / 养生馆老板 / 社区门店老板 / 泛健康人群
目标情绪：XX → XX（如：怀疑→相信）
知识库素材：（列出可用的具体素材）
```

**第四步：选题评审打分**
对每个通过合规预检的选题，按五个维度打分（每项1-10分）：

| 维度 | 评分标准 |
|------|----------|
| 点击欲望 | 看到这个选题/标题，目标人群想不想点进来？ |
| 接地气程度 | 村口大爷能不能看懂？有没有专业术语障碍？ |
| 招商转化潜力 | 看完能不能引导到分销合作？ |
| 科学可信度 | 有没有临床数据/专家/案例支撑？不能空口说 |
| 合规安全度 | 这个选题在写作和发布过程中踩线的风险有多大？ |

分级标准（五维度总分50分）：
- **S级**（总分≥42）：立刻可以写，预期效果好
- **A级**（34-41分）：值得写，可能需要微调角度
- **B级**（25-33分）：备选，有潜力但需要优化
- **淘汰**（<25分）：不输出

### 输出格式
输出10-15个选题，按S→A→B排列，每个选题附评分、分级和合规安全标记：
```
选题名称：
内容线：科学信任 / 健康科普 / 品牌故事 / 招商转化
核心角度：
目标人群：
目标情绪：XX → XX
知识库素材：
合规预检：✅通过（🟢安全 / 🟡谨慎 / 🔴高危）
评分：点击X + 接地气X + 招商X + 科学X + 合规X = 总分XX → S/A/B级
```

## 阶段二：标题生成（`/nsksd 标题 <选题>`）

### 输入
一个确定的选题（来自阶段一，或用户自定义）。

### 执行流程

为该选题生成5个标题变体，覆盖不同风格：

**标题公式库**（每次至少使用3种不同公式）：

1. **数字冲击型**：「X人临床数据证明：纳豆激酶不是玄学」
2. **反常识型**：「为什么日本89.4%的人吃纳豆，中国人却在交智商税？」
3. **恐惧驱动型**：「你的客户正在被直播间抢走」
4. **利益承诺型**：「社区门店如何靠一款产品锁定500个家庭？」
5. **提问引导型**：「养生馆老板：你还在靠手艺赚钱吗？」
6. **悬念型**：「4月24日杭州，纳豆激酶行业可能要变天了」
7. **对比型**：「卖纳豆激酶的公司千千万，为什么这家要自己花钱做临床？」

### 标题三原则

1. **村口大爷原则**：不用专业术语，说人话。"颈动脉内膜中层厚度"要翻译成"血管壁变厚"
2. **点击优先原则**：标题决定80%的打开率，宁可花1小时改标题也不要花1小时改内文
3. **人工复核原则**：AI出5个标题，人来挑选和修改。**标题不能100%交给AI**

### 标题合规检查（5项必查，每个标题都要过）

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 1 | 是否含医疗术语？ | 不得出现"治疗/治愈/疗效/药物/处方"等词 |
| 2 | 是否有绝对化用语？ | 不得出现"最""第一""唯一""根治""100%"等 |
| 3 | 是否暗示保证效果？ | 不得出现"保证有效""无效退款""立竿见影" |
| 4 | 是否使用恐吓诱导？ | 不得出现"不吃就晚了""再不注意就危险了" |
| 5 | 是否涉及虚假权威？ | 引用专家/机构必须真实可查 |

> 注意：标题是合规审查的第一道防线。标题违规=整篇文章违规，平台审核机器人首先扫描的就是标题。

### 标题禁用清单
- 禁止夸大疗效（"治好""根治""特效""神药"）
- 禁止使用"放大招""炸裂""颠覆""赋能""闭环"等AI味/营销号词汇
- 禁止假冒专家口吻（"医生建议""专家警告"——除非真的是该专家说的）
- 禁止虚假数据
- 禁止使用破折号（——），改用逗号或拆两句

### 输出格式
```
标题1：「xxx」（公式：数字冲击型）
标题2：「xxx」（公式：反常识型）
标题3：「xxx」（公式：提问引导型）
标题4：「xxx」（公式：利益承诺型）
标题5：「xxx」（公式：悬念型）

推荐：标题X（推荐理由）
```

## 阶段三：大纲生成（`/nsksd 大纲 <选题>`）

### 输入
一个确定的选题 + 可选的已选标题。

### 执行流程

1. 读取 `references/knowledge-base.md` 匹配相关素材
2. 按以下万能结构生成大纲：

```
钩子开头（100字以内，制造共鸣或冲突）
    ↓
行业洞察/痛点分析（建立"你懂他"的感觉）
    ↓
解决方案/趋势机会（给出方向）
    ↓
科学背书/数据支撑（建立信任）
    ↓
行动路径/案例验证（降低决策门槛）
    ↓
软性CTA（"了解合作"而非"立即购买"）
```

3. 大纲中每个部分标注：
   - 预计字数
   - 可用的知识库素材（具体到哪份文档的哪个数据点）
   - 写作注意事项

### 输出格式
完整的分段大纲，每段附素材来源和字数建议。总字数建议1500-2500字。

## 阶段四：全文撰写（`/nsksd 文案 <选题>`）

### 输入
一个选题 + 可选的标题 + 可选的大纲。如果没有提供标题和大纲，Skill会先自动执行阶段二和阶段三。

### 执行流程

**写作前：合规武装（必须在动笔前完成）**
1. 读取 `references/knowledge-base.md` 获取全部可用素材
2. 读取 `references/compliance.md` 加载完整合规审查机制（法律法规+平台规则+处罚案例）
3. 读取 `references/compliance-checklist.md` 加载20项检查清单
4. 确认该选题的合规安全分级（🟢/🟡/🔴），决定写作策略：
   - 🟢安全选题：正常写作，完成后走标准复审
   - 🟡谨慎选题：写作中每段都要对照禁用词表，完成后逐句复审
   - 🔴高危选题：写作中逐句检查合规，完成后全文逐项审查，建议人工二审

**写作中：合规护栏（写作过程中的7条铁律）**
1. **身份定位**：全文语境必须在"食品"范畴内，绝不越界到"药品/保健品"
2. **数据引用**：每个数据点必须可追溯到知识库原文，不虚构不夸大
3. **专家引用**：必须标注完整身份（姓名+职务+发言场合），不编造不泛化
4. **功效表述**：只用"辅助""养护""有助于""研究数据显示"，绝不用"治疗""治愈""预防"
5. **免责声明**：涉及功效的文章末尾必须加"本品为食品，不能替代药物。如有健康问题，请咨询专业医生。"
6. **禁用词实时排查**：写作过程中实时对照禁用词表（见compliance.md第四章），发现即替换
7. **平台适配**：根据目标发布平台（公众号/小红书/抖音）遵守对应平台的专属敏感词规则

5. 按大纲结构撰写全文（在合规护栏内创作）

### 写作风格要求

**核心原则：村口大爷能看懂**

- 口语化为王，像朋友聊天，不像AI写报告
- 永远具体：不说"很多客户"，说"80%的客户"
- 结论先行，再展开论据
- 短中长句交替（23%短句 / 50%中句 / 26%长句）
- 问号远多于感叹号（用追问代替感叹，用留白代替感慨）
- 适度暴露不确定："大概率""说不准""也许吧"

**签名句式**（自然使用，不要每篇都用全）：
- "你会发现" — 引导读者自己得出结论
- "为什么呢？" — 独立成段制造节奏
- "说白了" / "本质上" — 把复杂概念翻译成人话
- "大概率" — 概率性判断，不说绝对话

**禁用词**（写完必查）：
赋能、链路、飞轮、认知升级、颗粒度、抓手、闭环、触达、矩阵、范式、痛点、底层逻辑（单独使用）、护城河（泛化使用）、降维打击、生态位、心智模型

**禁用句式**：
「不是…而是…」「不是…是…」「并非…而是…」等对立否定句式——AI味极浓，改用直接陈述或用"说白了""本质上"引出判断。

**段落节奏**：
- 长文用 ◆ 或 ---- 切段，3-6个独立段落
- 编号列表（1️⃣2️⃣3️⃣）结构化复杂内容
- "……" 留白，话题切换或意犹未尽

**CTA规范**：
- 所有行动引导放在文末，不要在文章中间推销
- 用"了解合作"而非"立即购买"
- 用"想聊聊"而非"立即扫码"

### 数据引用规范

引用知识库数据时必须准确，常用数据点：

| 数据 | 正确表述 | 来源 |
|------|----------|------|
| 1062人临床 | "迄今纳豆激酶领域最大规模临床试验，1062人参与" | 科学循证文献合辑 |
| 斑块改善率 | "颈动脉斑块改善率66.5%-95.4%" | 临床册子 |
| 认知衰退风险 | "视觉空间功能衰退风险降低约65%" | 浙大RCT研究 |
| 专家建议 | "北京神经内科学会等3家学术机构、30余位专家" | 专家建议原文 |
| FU用量 | "10800FU/天（1062人试验）或8000FU/天（浙大RCT）" | 对应研究论文 |
| EFSA安全性 | "欧洲食品安全局2017年认证安全" | 科学循证文献 |

### 输出格式
完整文案（1500-2500字），包含标题、正文、CTA。文末附一段"素材来源清单"供团队核实。

## 阶段五：合规复审（`/nsksd 复审` 或 `/nsksd 合规检查`）

### 输入
一篇已写好的文案（来自阶段四或用户自己写的）。两个命令功能相同，`/nsksd 合规检查` 可独立于其他阶段使用。

### 执行流程

1. 读取 `references/compliance.md` 加载完整合规审查机制
2. 读取 `references/compliance-checklist.md` 加载20项检查清单
3. 按以下四大类、20项逐条检查：

---

### 第一类：法律红线（8项，违反即违法，罚款20万-100万）

| # | 检查项 | 法律依据 | 结果 |
|---|--------|----------|------|
| 1 | 是否将食品宣传为药品？ | 《广告法》第17条 | ☐ |
| 2 | 是否出现疾病治疗/预防承诺？ | 《广告法》第17条 | ☐ |
| 3 | 是否使用"最高级""国家级"等绝对化用语？ | 《广告法》第9条 | ☐ |
| 4 | 是否虚构/篡改临床数据？ | 《广告法》第4条 | ☐ |
| 5 | 是否利用患者形象做功效证明？ | 《广告法》第18条 | ☐ |
| 6 | 是否冒用或虚构专家身份？ | 《广告法》第11条 | ☐ |
| 7 | 食品描述是否涉及疾病预防治疗？ | 《食品安全法》第71条 | ☐ |
| 8 | 是否以学术论文名义做商业宣传？ | 《食品安全法实施条例》第38条 | ☐ |

> ⚠️ **铁律**：第一类任何一项不通过 = 整篇文案禁止发布，必须修改后重新审查。

### 第二类：平台规则（6项，违反即限流/封号）

| # | 检查项 | 平台依据 | 结果 |
|---|--------|----------|------|
| 9 | 公众号：是否触发健康类内容审核？ | 微信公众平台运营规范4.1 | ☐ |
| 10 | 公众号：是否含有引导外部交易的内容？ | 微信公众平台运营规范3.3 | ☐ |
| 11 | 小红书：是否含绝对化/夸大用语？ | 小红书《广告违禁词规范》 | ☐ |
| 12 | 小红书：是否触发医疗健康类限流词？ | 小红书内容审核规范 | ☐ |
| 13 | 抖音：是否符合健康科普内容创作者资质？ | 抖音《健康内容管理规范》 | ☐ |
| 14 | 抖音：是否含有"保健食品"相关违规表述？ | 抖音广告审核规范 | ☐ |

> 根据文案目标发布平台，选择对应平台的检查项。多平台发布则全部检查。

### 禁用词速查（复审时逐一扫描）

**绝对禁止（出现即违法）**：
治疗/治愈/根治 → "辅助健康管理""营养补充"
药物/药品/处方 → "功能性食品""营养食品"
疗效/药效 → "研究数据显示""临床观察发现"
预防XX病/防治 → "有助于维护XX健康"
最好/最强/第一/唯一 → "领先""优质""代表性"
100%有效/保证见效 → "研究显示XX%的受试者..."
无副作用/绝对安全 → "安全性数据良好""耐受性好"

**高风险（平台易触发审核）**：
降血压/降血脂/降血糖 → "有助于维护心血管健康"
溶栓/溶解血栓 → "纤溶酶活性研究"
抗癌/防癌 → 禁止使用，不做替换
消炎/杀菌 → "有助于维护身体平衡"
排毒/清肠 → "膳食营养补充"
增强免疫力 → "营养支持"
患者/病人 → "关注健康的人群""受试者"
临床验证/医学证明 → "研究数据显示""科学观察"
脉管焕新 → 禁止使用（同行已因此被央视点名）

**AI味/营销号词汇**：
赋能/链路/飞轮/认知升级/颗粒度/抓手/闭环/触达/矩阵/范式/底层逻辑/降维打击/生态位/心智模型/放大招/炸裂/颠覆

**禁用句式**：
「不是…而是…」「并非…而是…」→ 改用直接陈述或"说白了""本质上"引出判断

**平台专属敏感词**：
- 小红书："私聊""加微""详询客服"→"评论区留言"；"代理""加盟""拿货"→"合作伙伴计划"；"月入XX万""躺赚"→禁止
- 抖音："点击购物车""橱窗"→食品类需审核资质；"医生推荐""专家推荐"→需实名认证
- 公众号：外部链接（淘宝/京东）→可能被判定诱导外链；"限时优惠""最后XX名"→可能触发营销限流

### AI结构化预处理（排版前自动执行）

读取文案后检测Markdown结构完整度，如果缺少标题/加粗/列表等格式标记，自动补充：

1. **加标题**：识别逻辑段落转换点，插入 `##` 标题
2. **分段落**：确保段落间有空行，长段落在语义转换处拆分
3. **加列表**：识别并列/枚举内容，加列表标记
4. **加强调**：关键词、产品名、核心概念加 `**加粗**`
5. **清理格式**：去除多余空行、修正缩进、统一标点
6. **不改措辞**：只加结构标记，不调语序、不增删内容

## 关键提醒

1. **纳豆激酶是食品，不是药品**。任何内容都不能暗示治疗效果，必须使用"辅助""养护""改善风险"等表述
2. **NSKSD是具体品牌**，不等于"纳豆激酶"品类。知识库中的临床数据仅针对NSKSD产品，不能推广到所有纳豆激酶
3. **目标读者是门店老板**，不是消费者。内容要讲"赚钱逻辑"，不是讲"产品功效"
4. **村口大爷原则**是最高优先级的写作标准。任何时候在"专业"和"通俗"之间做选择，选通俗
5. **标题永远是第一优先级**。内容素材不缺，关键是"看到标题愿意点进来"
6. **排版走themes/**。写完文案后用 `scripts/format/format.py` 排版，默认主题newspaper，按内容线选择推荐主题
