# NSKSD 写作风格规范（独立版）

> 适用对象：日生研 NSKSD 纳豆激酶自媒体内容（公众号文章、飞书文档、选题文案）
> 版本：V9.9 · 2026-04-22
> 注意：本文件是 NSKSD 客户专属规范，与任何个人写作风格无关，不引用外部个人风格文档。
> V9.9 新增：段落字数硬上限 100 字 + 每篇 3-6 个 `##` 二级小标题（解决客户端"大段文字可读性差"反馈）

---

## 1. 读者画像

| 维度 | 描述 |
|------|------|
| 主要读者 | 社区门店老板、美容院老板、养生馆老板（潜在分销商）|
| 终端用户 | 40-60 岁三高人群（高血压、高血脂、高血糖）|
| 教育背景 | 中学至大专，理解科普信息但不看学术论文 |
| 阅读习惯 | 微信公众号、朋友圈、视频号，阅读时间碎片化 |
| 关注点 | 产品效果、赚钱逻辑、操作门槛、合规风险 |

---

## 2. 语感与语气

- **大白话为主**：把专业概念翻译成生活语言，让 40-60 岁读者第一遍就看懂
- **专业讲解并重**：数据和机制要说清楚，不模糊，不含糊其词
- **有态度，有判断**：不做信息搬运，要有立场和视角
- **结论先行**：先给结论，再展开论据，不绕弯子

### 人称规范

| 场景 | 用法 |
|------|------|
| 对读者 | "您"（尊称，不用"你"）|
| 品牌自称 | "我们日生研"或"日生研"（不用"小编"、"我们"）|
| 案例人物 | 第三人称具名（老王、王阿姨、李老板）|
| 数据来源 | 直接标注机构全称，不用"某研究机构"模糊处理 |

## 3. 标题写法

- 直截了当，不堆砌修饰语
- 可以分两行，像说话一样自然
- 数字具体："1062 人"比"上千人"有力
- 问句优于感叹句

**好例子：**
- 「每天一粒，浙大追踪1062人两年」
- 「养生馆卖保健品为什么总被客户嫌贵？」
- 「颈动脉斑块能消吗？这个数据值得看」

**坏例子（不要用）：**
- 「震撼！日本研究揭示颠覆性认知」（一惊一乍 + 禁用日本表述）
- 「深度好文：纳豆激酶护城河解析」（禁用词）
- 「不是卖产品，而是卖认知」（禁用句式）

## 4. 正文节奏与结构（V9.9 硬约束）

### 段落分段铁律

- **单段不超过 100 字**（约 3-4 行）。超过必拆
- **两句一段**是常态，一句一段也可以（强调句 / 问句 / 数据句）
- 长句内部用逗号拆，句与句之间用句号断，能句号就不要逗号凑成长段
- 短中长句交替（短句 ≤ 15 字，长句 ≤ 50 字）
- 段落之间空一行，不用 `----` 分隔符堆节奏

### 小标题分章铁律

**每篇文章必须有小标题**，这是可读性的脊梁。

- 二级标题用 `##`（3-6 个，对应 6 段式结构中的主干段）
- 三级标题用 `###`（可选，长段落里再切视角时用）
- 小标题**不是装饰，是路标**：让读者扫标题就能知道这段讲什么
- 小标题本身就是完整的观点，不写成"一、二、三"式目录

**好的小标题示例：**
```
## 浙大追踪 1062 人，两年的结论
## 为什么颈动脉斑块最怕它
## 养生馆老板怎么把这个故事讲给客户
## 三高人群最关心的三个问题
```

**坏的小标题（不要用）：**
```
## 背景介绍
## 研究方法
## 总结
```

### 换角度规则

- 每 200-300 字换一个角度或话题，换角度必起新小标题
- 编号列表（1. 2. 3.）整理复杂信息，三条以内不用列表

## 5. 数据使用规范

数据是说服力的核心，但必须真实。

**每个数据点必过三问：**
1. 这个数据有来源 URL 吗？
2. URL 还活着吗（WebFetch 验证）？
3. 有第二个来源交叉印证吗？

**找不到来源时的降级写法：**
- 有行业通识但无具体来源 → 写"行业内普遍认为……"
- 数据来自内部资料无法公开 → 写"根据内部追踪数据"
- 完全无法核实 → 直接删掉，不编造

**禁止行为：**
- 捏造数据
- 捏造 URL
- 捏造机构名称
- 用"据报道""有研究发现"替代真实来源

详细验证流程参见：`references/data-verification.md`

## 7. 禁用句式

| 禁用类型 | 示例 | 原因 |
|----------|------|------|
| 对立否定 | "不是…而是…" | AI 味极浓，生硬 |
| 并非否定 | "并非…而是…" | 同上 |
| 强调转折 | "不仅…而且…" | 套话 |
| 破折号滥用 | 一篇文章超过 2 个破折号 | 打乱语流 |

**改法：直接陈述，或用"说起来""讲到这里""换个角度看"做过渡。**

## 8. 合规硬约束（不能碰的红线）

- **禁止使用"日本"相关词汇**对外传播：日本/日式/日系/东瀛/和风/中日/日企/日资/日货/日产
- **标准替代**：原研方 / 国际合作研发机构 / NSKSD纳豆激酶 / 国际研究
- **叙事主场**：科学循证为主（1062 人临床 + 浙大 RCT + 中国专家共识），不用"异域光环"
- **豁免**：产品包装、法律标签、学术论文原文引用（面向监管/法务/学术溯源场景）
- **功效宣称限制**：不说"治疗"，说"有助于""研究发现"；不承诺效果，说"个体存在差异"

## 8B. 引用来源规范（V10.3 新增硬约束）

客户反馈：模型写文章喜欢在每段后面加「（来源：同上）」或用「袁总/老袁」这种亲昵称谓代替人的原名。这两种都不合规。

### 规则一：来源必须用**人的原名**，禁止亲昵/尊称

| ❌ 禁用 | ✅ 正确 |
|--------|--------|
| （来源：袁总采访） | （来源：袁岳访谈实录） |
| （采访：老袁） | （采访：袁岳） |
| （讲述：张老师） | （讲述：张伟东博士） |
| —— 陈董 | —— 陈文明 |
| 王姐说 | 王海燕说 |

**白名单**：姓 + 完整名（至少 2 字名） + 可选头衔（教授/博士/主任医师/研究员）。例如 "张文宏教授" 合法。

**黑名单词**：总、哥、姐、老师（单独作后缀）、董、爷、叔、姨；老X / 小X（老袁、小王）。

### 规则二：每次引用都要**写清具体来源**，禁止"同上/同前/见上"

一篇文章每个数据/引文都独立具名，哪怕重复也要写全——读者不一定从头读到尾。

| ❌ 禁用 | ✅ 正确 |
|--------|--------|
| （来源：同上） | （来源：《中国心血管病报告 2024》） |
| （来源：同前） | （来源：浙江大学 1062 人 RCT 研究 DOI:10.xxxx） |
| （来源：见上） | （来源：NSKSD 官网「临床研究」页） |
| （来源：同 1） | （来源：日生研 2024 蓝皮书参编数据） |

### 规则三：括号统一用**全角**`（来源：XXX）`

半角 `(来源: XXX)` 会被 `citation_check.py` 拦截退回——因为 `format.py` 的"自动灰化"规则只认全角模板。用全角的好处：读者在推文里看到的来源标注自动变浅灰色，和正文拉开视觉层级。

### 自动化门控

`scripts/citation_check.py` 扫三项：
- 亲昵称谓出现在「括号来源内」或「—— / 采访者 / 摘自」行
- 来源模糊回指（同上/同前/见上/同 N）
- 半角括号来源（`(来源：...)`）

任一命中 → `trigger.status = rejected_citation_check`，不发布。

```bash
python3 scripts/citation_check.py artifacts/<SID>/step3-article.md
# 退码 0=通过 / 1=违规 / 2=文件错
```

## 9. 写作模板（按场景）

### 科普文开头（建议写法）
```
[具体数字/现象] + [读者熟悉的场景] + [悬念/疑问]

例：
"浙江大学连续追踪 1062 人，整整两年。
最后的数据让研究团队也没想到——"
```

### 案例故事开头
```
[人物 + 职业 + 具体时间] + [遇到的问题] + [转折]

例：
"老王开养生馆 6 年，每次推保健品都碰壁。
直到去年 11 月，一个客户问了他一个问题……"
```

### 数据呈现
```
[数据] + [来源] + [对读者意味着什么]

例：
"临床数据显示，连续服用 26 周后，颈动脉内膜中层厚度
平均下降 0.22mm（来源：Journal of Cardiovascular Risk，
2023年，DOI: 10.xxxx）。
对三高人群来说，这个变化意味着……"
```

## 10. 自检 checklist（发布前过一遍）

```bash
# 快速检查禁用词（在文章目录下运行）

# 检查禁用日本表述
grep -n "日本\|日式\|日系\|东瀛\|和风\|中日\|日企\|日资\|日货\|日产" 文章.md

# 检查禁用句式
grep -n "不是.*而是\|并非.*而是\|不仅.*而且" 文章.md
```

**发布前确认清单：**
- [ ] 读出来像日生研品牌在说话，不像 AI 在生成
- [ ] 感叹号少于 3 个（一整篇）
- [ ] 没有禁用词
- [ ] 没有禁用句式
- [ ] 每个数据都有可验证来源或降级处理
- [ ] 没有"日本"相关词汇（豁免场景除外）
- [ ] 人称统一（"您"为主，品牌自称"我们日生研"）
- [ ] 结论在开头或第一段
- [ ] 收尾留有余地，不下绝对结论
